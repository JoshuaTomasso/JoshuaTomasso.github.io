Overview

FPS Zombie is a first-person zombie survival game where players enter a town-based environment, collect weapons, and defend themselves against zombies that aggressively track the nearest player. The core objective is simple: survive as long as possible.

The project includes both single-player and multiplayer flows, along with gameplay systems for movement, shooting, pickups, health, death, UI, audio feedback, and networked player support.

  • First-person gameplay programming in Unity using C#
  • Combat and weapon interaction systems
  • Enemy pursuit and survival loop design
  • HUD and player feedback systems
  • Multiplayer architecture using Unity Netcode for GameObjects
  • Lobby and Relay integration for session creation and joining

What I Worked On

Network Camera Positioner

I implemented the networked camera positioning setup for player-owned characters.

  • The camera is attached to the player prefab with the camera and audio listener initially disabled.
  • When the network object spawns, the camera and audio listener are enabled only for the owning player.
  • During Update(), the camera position and rotation are continuously aligned to the local player's transform.

This ensures each player only controls and hears from their own perspective in multiplayer, which is a core requirement for a networked FPS.

Network Camera Controller

I built the local player camera controller logic for owner-only input.

  • Locks and hides the cursor for the owning player at startup.
  • Reads Mouse X and Mouse Y input to drive first-person camera rotation.
  • Unlocks and shows the cursor while the game is paused.
  • Runs only for the owning client to prevent remote players from controlling the local camera.

This cleanly separates local input authority from replicated player objects, which is important in multiplayer game architecture.

Zombie Game Multiplayer

I worked on the main multiplayer manager using Netcode for GameObjects.

  • Lives on an empty scene object and persists with DontDestroyOnLoad.
  • Handles the distinction between single-player and multiplayer startup flow.
  • Starts the host/server path for single-player and loads into lobby flow for multiplayer.
  • Sets up join checks, player data, and server-related delegates used by other systems.

This system coordinates the core networking flow and ties together scene transitions, host/client startup, and shared session behavior.

Zombie Game Lobby

I built the lobby system using Unity Lobby and Unity Relay.

  • Creates public or private lobbies using an entered name.
  • Sets up Relay so players can connect without manual networking configuration.
  • Supports Quick Join, Join with Code, and Join with Lobby ID.
  • Filters and lists available lobbies that still have open slots.
  • Maintains periodic lobby refresh and heartbeat updates so lobbies remain visible and active.

This shows practical experience with modern multiplayer services and session UX, which is a strong portfolio signal for gameplay and network roles.

Network Zombie Prefab

I worked with a zombie prefab setup where logic and colliders are kept separate from the visual model.

  • Gameplay logic and collision are separated from the visual model hierarchy.
  • Improves flexibility and keeps gameplay systems cleaner.
  • Supports easier iteration on enemy visuals or animation without touching core logic.

Separating logic from visuals is a practical prefab design habit that pays off during iteration and debugging.

Notes

Gameplay Loop

Players spawn into the map and must quickly find and use weapons to survive incoming zombie attacks. After short intervals, zombies begin running toward the closest player, creating constant pressure and forcing players to balance movement, aim, ammo, and survival decisions. When a zombie is killed, it can randomly drop health or ammo, adding a resource-management layer to the combat loop.

  1. Explore the map and pick up weapons
  2. Fight off approaching zombies
  3. Manage ammo and reload timing
  4. Recover health and supplies from drops
  5. Respawn and re-enter the fight after death
